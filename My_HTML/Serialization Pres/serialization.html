Serialization Overview Implementation Q&A

Overview


  • To serialize an object means to convert its state into a byte stream in a way that always it to be reverted back into an object.
  • Deserialization is the process of converting the serialized form back into a copy of the object.
  • When an object is serialized, the class definition itself is not recorded. It is the responsibility of the system to locate the necessary class files.
  • To create a serializable object, simply implement Serializable
  • Declare and initialize a field static final long serialVersionUID = 1L (Strongly recommended)
  • Use an ObjectOutputStream to serialize an object
  • Use an ObjectInputStream to deserialize an object

Implementation


Creating the class


    public class Example implements Serializable {
    
        public static int count;
        public String name;
        public transient int ssn;
        
        private static final long serialVersionUID = 1L;
    }

Serializing the object


    String filename = "file.txt";
    Example ex = new Example();
    try (FileOutputStream   fos = new FileOutputStream(filename);
         ObjectOutputStream oos = new ObjectOutputStream(fos)) {
        
        oos.writeObject(ex);
        
    } catch (IOException e) {
        e.printStackTrace();
    }

Deserializing the object


    String filename = "file.txt";
    Example exCopy;    
    try (FileInputStream   fis = new FileInputStream(filename);
         ObjectInputStream ois = new ObjectInputStream(fis)) {
        
        exCopy = (Example) ois.readObject();
        
    } catch (ClassNotFoundException e) {
        e.printStackTrace();
    } catch (IOException e) {
        e.printStackTrace();
    }

Q&A


How many methods Serializable has? If no method then what is the purpose of Serializable interface? java.io.Serializable doesn't have any method and is called Marker Interface. Creating a class that implements Serializable gives the compiler an indication that it is possible serialize this object.
What is serialVersionUID? What would happen if you don't define this? SerialVersionUID is used for version control. The default value of serialVersionUID is calculated based on the object structure. This value can be affected by the compiler, so it is best to explicitly define a value.
While serializing you want some of the members not to serialize? How do you achieve it? All members of an object will be serialized unless it is marked transient or it is a static member.
What will happen if one of the members in the class doesn't implement Serializable interface? If you try to serialize an object of a class which implements Serializable, but the object includes a reference to an non-serializable class then a NotSerializableException will be thrown at runtime.
Suppose super class of a new class implement Serializable interface, how can you avoid new class to being serialized? If a class' parent already implements Serializable interface, then it is already Serializable. Since you cannot unimplemented an interface, to avoid Java serialization, you need to implement the writeObject() and readObject() methods to throw a NotSerializableException.
Which methods are used during Serialization and Deserialization in Java? To store any object via serialization, call ObjectOutputStream.writeObject(someObject). To deserialize that object, call ObjectInputStream.readObject().
