Reflection in Java

What is reflection?

Reflection is the act of examining code at runtime.

More specifically, reflection allows us to check the classes, fields, methods, and constructors of objects at runtime.

We can store these things in objects whose datatypes are named for what they model, e.g., methods are stored in Method objects.

What are the downsides?

  • Slows down performance
  • May Not Run Under Security
  • Exposure of Internals

The java.lang.reflection Package

Class Purpose
Class Models the class of an object or primitive
Field Models the data members of a class. Can access public and private members
Method Models methods. Has an invoke() method for calling methods. Can also pass parameters.
Constructor Models constructors, including names, modifiers, and parameters
