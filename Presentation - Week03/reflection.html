Reflection in Java

What is reflection?

Reflection allows us to examine code at runtime.

That means we can see the classes, fields, methods, and constructors of objects at runtime.

For example, if I have an object of an unkown type, I can use the getClass() method to discover its type.

Reflection is a powerful tool that can allow us to perform operations otherwise impossible, but it is a very advanced feature that should only be used by experienced programmers.

Important Reflection Classes

Class Purpose
Class Models the class of an object or primitive
Field Models the data members of a class. Can access public and private members
Method Models methods. Has an invoke() method for calling methods. Can also pass parameters.
Constructor Models constructors, including names, modifiers, and parameters

What are the downsides?

  • May not run under a security manager
  • Can expose private members
  • Slows down performance
