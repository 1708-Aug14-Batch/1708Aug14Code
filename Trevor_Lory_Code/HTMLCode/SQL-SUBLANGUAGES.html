SQL SubLanguages

All commands of sql fall in to one of the following categories

DDL (Data Definition Language)

These statements are used to create tables and databases and define field properties or table properties

The various objects that a database can have are table, view, synonym, sequence, index and type

DDL provides the following statements to manipulate these objects:

CREATE ALTER DROP TRUNCATE RENAME
  • Used to create tables, procedures, and functions
  • Used to change the properties of tables
  • Used to remove a table from the database or place it inside the recycle bin
  • Used to remove all rows from a table
  • Used to rename a table, view, or a sequence

DML (Data Manipulation Language) aka CRUD

The statements that falls under this category are used to update data or add or remove data from tables

INSERT UPDATE SELECT DELETE
  • Adds one or more rows to a table
  • Updates values within a table
  • Queries Data from tables and views to be viewable or assignable
  • Deletes rows within a table

DCL (Data Control Language)

It is used to control who access the data

DCL provides 2 statements for access control:

GRANT REVOKE
  • Gives privledges to edit or view the database
  • Takes away privledges to edit or view the database

TCL (Transaction Control Language)

After any DML statement is executed, the updated data is not made persistent in the respective tables.

In order to save the data it has to be explicitly saved by the user using a TCL command.

The user can either save or discard it. The statements provided by TCL for this purpose are:

SAVEPOINT ROLLBACK COMMIT
  • A saved state in the ongoing execution that can be rolled back to
  • Undos most commands so as to revert to a previous state within the database
  • Makes permanent any changes done before it

DQL (Data Query Language)

This is to retrieve data from sql server without changing the data itself

Is also called the Data Retrieval Language (DRL)

SELECT ORDER BY GROUP BY
  • Queries Data from tables and views to be viewable or assignable
  • Orders data in ascending or decsending data based on a inputed column
  • Groups data together within a table so operations can either be done to them or for them to be ordered differently
