The Java Virtual Machine


The Java Virtual Machine (JVM) can be used to refer to three separate things...

  • The abstract specification of the JVM. This is analogous to a java interface.
  • A concrete implementation of the JVM. This ia analogous to a java non-abstract class.
  • A runtime instance of the JVM. This is analogous to an instance of a java class.

There are many different implementations of the JVM, though each one conforms to the abstract specification of a JVM.

Most programming languages convert source code directly into machine code. Jave converts source code into its own machine code known as bytecode. This runs on a virtual machine which integrates system calls with the unique operating system it is run on.

Since the Java Virtual Machine chooses how to interpret the bytecode (based on the Operating System of the machine it is on), it can run the same set of bytecode on any machine. This is how java code is Write Once, Run Anywhere.

Different JVMs are allowed flexibility in how they handle things like...

  • Garbage collection: A JVM is not technically required to implement garbage collection at all-- this is why garbage collection cannot be forced.
    • The JVM implements garbage collection by running threads known as daemon thread. Daemon threads are used by the JVM itself.
  • Thread scheduling: Depends heavily on the OS threading and as such it is loosely restricted by the JVM.

The greatest advantage that the JVM gives us is encapsulating the Operating System's implementation of machine code. Without that Java would be platform-dependant.

The JVM has limited support for booleans, so most compilers will convert booleans into ints or bytes before creating bytecode.

The JVM uses an extra primitive type which is unavailable to a Java programmer: the returnAddress type. This primitive is used to implement finally clauses.

The JVM is a 32-bit virtual machine.

byte 8-bit signed two's complement integer (-2^7 to 2^7 - 1, inclusive)

long 64-bit signed two's complement integer (-2^63 to 2^63 - 1, inclusive)
char 16-bit unsigned Unicode character (0 to 2^16 - 1, inclusive)
float 32-bit IEEE 754 single-precision float
double 64-bit IEEE 754 double-precision float
