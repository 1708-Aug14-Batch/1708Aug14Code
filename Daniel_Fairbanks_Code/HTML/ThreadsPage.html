What are threads?


A thread is a single line of execution.


What is multithreading?


Multithreading is running multiple threads in concurrency.


Why use multithreading?


  • Parallelzation - Systems can use multithreading to split data and tasks into parallel subtasks and let the underlying architecture manage how the threads run, either concurrently on one core or in parallel on multiple cores.
  • Faster execution - this advantage of a multithreaded program allows it to operate faster on computer systems that have multiple central processing units (CPUs) or one or more multi-core processors.
  • Responsiveness - Multithreading can allow an application to remain responsive to input. In a one-thread program, if the main execution thread blocks on a long-running task, the entire application can appear to freeze. By moving such long-running tasks to a worker thread that runs concurrently with the main execution thread, it is possible for the application to remain responsive to user input while executing tasks in the background.
    • Examples:
  • On a web server if a client is slow other clients can still communicate using other threads.
  • In a game program you can use threads for different tasks: AI, Graphics, User Input, Networking
