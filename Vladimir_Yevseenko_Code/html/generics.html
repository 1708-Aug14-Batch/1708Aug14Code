Generics in Java

What are generics?

Generics are a language feature implemented in Java 5 that allow for parametrization of classes, interfaces, and methods.

What was Java like before generics?

Prior to generics, classes simply dealt directly with the Object type and did the proper casting required to expose the actual type. For example, collections would return Object and accept Object as an argument. This makes runtime errors very common because you never know what your collection truly contains without using many instanceof conditionals to check. Also leads to a lot of ambiguity over what your methods actually do and what their arguments are.

Why are generics useful?

Runtime bugs are much harder to find than compile time bugs. Generics reduce a lot of common runtime errors to compile time errors due to the enforcement of the generics type system. Arguments, return types and parametrized classes now follow specific input and output patterns described by the information provided inside the <> brackets.

How does it actually work? Type Erasure!

These parametrized classes, interfaces and methods only exist before your program is compiled. The compiler replaces all type parameters used with the bounds the types require or just the Object type if there are no bounds. It also provides all the casts required for your program to compile once all the parametrized types are erased.

Things you cannot do!

  • You cannot use subtyping with generics
  • You cannot create generic arrays
  • Generics parameters cannot be primitives
